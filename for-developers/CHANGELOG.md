## [1.6.1] - 2026-04-22

### 🔧 Fixed - Hiroマーカー2枚同時認識の設定修正

2枚のHiroマーカーを並べても片方しか認識されない問題を修正。

#### 原因

AR.jsは デフォルトで `maxTrack: 1`（1マーカーのみ追跡）に設定されている。
**同じプリセット（Hiro）のマーカーを2枚同時に認識するには明示的に設定が必要**。

#### 修正内容

1. **AR.js設定に `maxTrack: 2` を追加**
   - a-scene の `arjs` 属性に明示指定
   - `trackingMethod: best` も追加して精度向上

2. **マーカー認識状況バー を追加**
   - 画面上部に **[左 ● 認識中]** [右 ● 認識中]** のインジケーター表示
   - 緑のドットが点滅 = 認識中
   - グレー = 未認識
   - どちらが認識されているか一目で分かる

3. **マーカー認識のコツをマーカー印刷ページに追記**
   - マーカー間隔: **30cm 以上**
   - カメラ距離: **50〜80cm**
   - 両方が画面内にはっきり映ること

#### 関連ドキュメント更新

- `CHANGELOG.md` - v1.6.1 リリースノート追加
- `マーカー印刷.html` - 2枚認識のコツを追記

## [1.6.0] - 2026-04-22

### 🛡️ Added - クラッシュ防止ガード（Stability Release）

予期しないエラーで画面が真っ白になることを防ぐ、**全体的な防御層** を追加。

#### 新規ファイル: `error_guard.js`

プロジェクト全体で共有する **グローバルエラーハンドラ**。
全ての主要HTMLページの最初に読み込まれる。

**機能**:
1. **JavaScriptエラーの捕捉**
   - `window.onerror` / `unhandledrejection` を処理
   - 画面が真っ白にならないよう、優しい通知パネルを表示
   - 連鎖エラーを防ぐため通知は1回まで

2. **localStorage セーフラッパー (`window.safeStorage`)**
   - `getJSON()` / `setJSON()` / `get()` / `set()` / `remove()`
   - JSON破損時は自動で削除してデフォルト値を返す
   - プライベートモードで localStorage 禁止時も落ちない

3. **画像読み込み失敗時のフォールバック**
   - MutationObserver で全ての `<img>` に自動で `onerror` を登録
   - 画像URL失敗時は SVG のプレースホルダに自動切替
   - 404エラーで画面が崩れない

4. **セーフDOM取得 (`safeGetEl` / `safeQuery`)**
   - null-safe なラッパー

5. **プロジェクトリセット機能 (`window.__resetProject()`)**
   - localStorage を全クリアして再読込
   - トラブルシューティングページにボタン配置

#### 各HTMLの防御強化

**全HTMLに `error_guard.js` を組み込み**
- `選択支援メニュー.html`
- `WebAR本番.html`
- `マーカー印刷.html`
- `運用マニュアル.html`
- `技術仕様書.html`
- `トラブルシューティング.html`
- `WebAR使い方ガイド.html`
- `index.html`

**menu_data.js 読込失敗時のインラインフォールバック**
- 最低限3メニュー（ラーメン・カレー・おすし）のハードコードデータで動作継続
- CDN接続不可時にも画面が崩れない

**app.js の防御的リファクタ**
- すべての主要関数を try-catch で保護
- DOM 要素取得に null チェック
- `window.safeStorage` を利用した堅牢な永続化
- 初期化失敗時もページは最低限動作

**WebAR のエラー復帰**
- ライブラリ読込タイムアウト時: ボタンが「🔄 再読み込みして再試行」に変化
- startAR のエラーもキャッチして画面クラッシュ回避

#### トラブル時の対応フロー

1. **軽微なエラー**: 画面右下に黄色い通知 → 「閉じる」で隠せる
2. **再読み込みで解決可能なエラー**: 通知内に「🔄 再読み込み」ボタン
3. **設定の不整合**: トラブルシューティングページの **「🔄 設定をリセット」** ボタン

#### 関連ドキュメント更新

- `CHANGELOG.md` - v1.6.0 リリースノート追加
- `トラブルシューティング.html` - リセット機能のセクション追加
- `index.html` - トラブル対応カードの説明に「システムリセット機能」を追記

メニュー選択時に **1個・2個・3個のどれでも** 選べるように改善。

- 3個未満選ぶと、足りない分が自動で埋められていた

- **2個選択**: 2品並んで表示される（4案すべて対応）

- モーダルのヒント文: 「最大3つ」→ **「1〜3つ、好きな数でOK」**

  - 1個: 中央に大きく表示
  - 2個: 左右に2カラム
  - 3個: 3カラム

#### WebAR側の対応

- 選択1品の場合: **左マーカーのみ表示**（右は「➕ 右を追加」として差し替え可能）
- 選択2品の場合: 左右に配分
- 選択3品の場合: 左右に配分（3品目は「↻変更」の候補）

## [1.2.3] - 2026-04-22

### 🔧 Fixed - マーカー認識を公式パターンに切り替え

**重要な修正**: AR.jsのバーコードマーカーの認識精度を大幅向上。

#### 問題の原因

- 自作SVGパターンがAR.jsの仕様と合っていなかった
- デフォルトの `3x3` マトリクスは **Hamming distance 0** で公式ドキュメントにも「実質使用不可」と記載されていた

#### 解決策

1. **AR.js公式のバーコード画像集を採用**
   - リポジトリ: `nicolocarpignoli/artoolkit-barcode-markers-collection`
   - すべて動作確認済みの公式画像（PNG）

2. **マトリクスタイプを `3x3` → `3x3_HAMMING63` に変更**
   - Hamming distance **3**（高い認識精度）
   - 最大8種類のマーカー（#0〜#7）が利用可能
   - 誤認識が激減

3. **マーカー値の再割り当て**
   - 🍜 ラーメン: Hiroプリセット（変更なし）
   - 🍛 カレー: **Hamming63 #3**（公式画像）
   - 🍣 おすし: **Hamming63 #5**（公式画像）

## バージョニング方針

**X.Y.Z** の3段階:

- **X (Major)**: 互換性のない変更（メニュー構造の刷新など）
- **Y (Minor)**: 後方互換性のある機能追加
- **Z (Patch)**: バグ修正・小さな改善

例:
- `1.0.0` - 初回リリース
- `1.1.0` - 読み上げ機能追加
- `1.1.1` - バグ修正
- `2.0.0` - メニューデータ構造の変更
